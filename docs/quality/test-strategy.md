# Test Strategy

## 1. Мета
Забезпечити перевірку функціональних і нефункціональних вимог системи EventMaster.

## 2. Рівні тестування
| Рівень | Призначення |
|---|---|
| Unit | Перевірка сервісів, валідаторів, утиліт |
| Integration | Перевірка взаємодії API, БД, репозиторіїв |
| System | Перевірка наскрізних сценаріїв |
| Acceptance | Перевірка готовності системи за бізнес-вимогами |

## 3. Покриття вимог
- FR-01, FR-02 — тести реєстрації та логіну
- FR-03, FR-04 — тести отримання списку та деталей подій
- FR-05 — тести бронювання та перевірки доступних місць
- FR-06, FR-07, FR-08 — CRUD тести для подій
- NFR-03, NFR-04 — тести безпеки та зберігання паролів

## 4. Підхід до тестування
- Unit-тести запускаються при кожній зміні бізнес-логіки.
- Integration-тести запускаються перед злиттям у main.
- Traceability Matrix використовується для перевірки, що кожна вимога має тестове покриття.

## 5. Критерії входу/виходу
### Вхід
- Вимоги зафіксовані в SSD.
- Архітектура описана в SDD.
- Доступні тестові дані та середовище.

### Вихід
- Усі критичні тести успішні.
- Немає blocker/critical дефектів.
- Матриця трасування актуальна.
