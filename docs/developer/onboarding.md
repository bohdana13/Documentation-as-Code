# Developer Onboarding

## 1. Мета документа
Допомогти новому розробнику швидко зрозуміти структуру проєкту EventMaster та правила роботи з документацією.

## 2. Перші кроки
1. Клонувати репозиторій.
2. Перейти в гілку розробки.
3. Ознайомитися з `docs/README.md`.
4. Прочитати SSD, SDD, ISD.
5. Запустити проєкт локально.

## 3. Правила роботи з документацією
- Не створювати документацію у Word/PDF як основний формат.
- Оновлювати Markdown-документи разом із кодом.
- Використовувати посилання на ID вимог з SSD.
- Не дублювати вимоги в кількох документах без потреби.

## 4. Мінімальний workflow
1. Створити feature branch.
2. Внести зміни в код.
3. Оновити документацію.
4. Запустити тести.
5. Створити pull request.
