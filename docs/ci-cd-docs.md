# Documentation CI/CD Pipeline

Цей документ описує процес автоматизації документації в межах CI/CD для проєкту EventMaster.

## 1. Артефакти для генерації
Наступні артефакти підлягають автоматичній генерації під час виконання pipeline:
- **API-документація:** Swagger UI на основі специфікації OpenAPI.
- **UI-документація:** Статична збірка Storybook.
- **Загальний сайт документації:** MkDocs-сайт (включає SSD, SDD, ISD тощо).

## 2. Тригери генерації
- **Push у гілку `feature/*`:** Запускає лише етап валідації (linting) для перевірки коректності Markdown та OpenAPI.
- **Merge у гілку `main`:** Запускає повний цикл (збірка, валідація, публікація).

## 3. Послідовність кроків pipeline
1. **Checkout:** Завантаження актуального коду репозиторію.
2. **Validate OpenAPI:** Перевірка файлу `openapi.yaml` на відповідність стандарту OpenAPI 3.0 за допомогою лінтера (наприклад, Spectral).
3. **Build Swagger UI:** Генерація статичних файлів для перегляду API.
4. **Build Storybook:** Компіляція UI-компонентів у статичний сайт.
5. **Build MkDocs:** Генерація загального сайту документації (`mkdocs build`).
6. **Publish:** Деплой згенерованих файлів.

## 4. Місце публікації
Документація публікується на **GitHub Pages**.
- Структура URL:
  - Головна сторінка: `https://<username>.github.io/eventmaster/`
  - API Swagger: `https://<username>.github.io/eventmaster/api/`
  - Storybook: `https://<username>.github.io/eventmaster/ui/`

## 5. Політика Versioning документації
- Версія документації строго відповідає версії продукту (наприклад, v1.0.0).
- Зворотньо несумісні (Breaking) зміни API вимагають створення нової мажорної версії документації.
