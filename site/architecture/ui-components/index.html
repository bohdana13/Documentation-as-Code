UI Components Documentation (Storybook)

Цей документ описує стани ключових UI-компонентів системи EventMaster для бібліотеки Storybook.

1. Компонент: EventCard

Компонент відповідає за відображення короткої інформації про подію у загальному списку.

Stories (Стани):

  • Default (Базовий стан): - Відображає назву події, дату, локацію та кількість доступних місць.
  • Кнопка "Забронювати" активна.
  • Sold Out (Альтернативний стан): - Відображається, коли availableSeats === 0.
  • Картка візуально затінена (opacity: 0.7).
  • Кнопка бронювання замінена на червоний текст "Місць немає".
  • Loading (Стан завантаження): - Відображається під час отримання даних з бекенду.
  • Використовує skeleton loader замість тексту та зображень.

2. Компонент: BookingButton

Інтерактивна кнопка для взаємодії з API бронювання.

Stories (Стани):

  • Default: Кнопка активна, синього кольору з текстом "Забронювати".
  • Disabled: - Кнопка сірого кольору, не реагує на клік.
  • Використовується, якщо користувач не авторизований або місць більше немає.
  • Loading: - Замість тексту відображається анімація спінера.
  • Кнопка тимчасово блокується (disabled) для запобігання подвійному надсиланню POST-запиту.
