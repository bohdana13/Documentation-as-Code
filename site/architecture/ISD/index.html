ISD — Infrastructure Specification Document

1. Середовища

Середовище Призначення
Local Локальна розробка та тестування
Test Перевірка перед релізом
Production Робоче середовище

2. Технологічний стек

  • Backend: Spring Boot / Node.js API (залежно від реалізації ЛР-7)
  • Database: MongoDB або інша документно-орієнтована БД
  • Frontend: Web UI
  • Documentation: Markdown + MkDocs
  • Version Control: Git

3. Інфраструктурні вимоги

  • Застосунок повинен запускатися через стандартний процес збірки.
  • Конфігурація середовища зберігається окремо від коду.
  • Секрети не зберігаються у відкритому вигляді в репозиторії.
  • Документація збирається як статичний сайт.

4. CI/CD

Мінімальний pipeline: 1. checkout репозиторію; 2. збірка застосунку; 3. запуск тестів; 4. перевірка структури документації; 5. генерація статичного сайту документації.

5. Структура репозиторію

project-root/
├── src/
├── docs/
├── tests/
├── mkdocs.yml
└── README.md

6. Резервування та актуальність

  • Документація зберігається в Git.
  • Кожна версія документації прив’язується до релізу застосунку.
  • Для змін у вимогах та архітектурі створюються окремі commit/pull request.
