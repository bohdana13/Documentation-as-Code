Definition of Done (Documentation)

Цей документ формалізує критерії готовності (DoD) для змін у проєкті EventMaster з точки зору актуалізації документації.

1. Правила для типів змін

Тип зміни Обов'язкові дії Перевірка (CI/Manual)
Додавання нового API-ендпоінта Оновити файл openapi.yaml (додати path, parameters, responses). Успішна валідація OpenAPI схеми в CI.
Зміна структури response/request Оновити існуючі схеми (components/schemas) в openapi.yaml. Swagger UI генерується без помилок.
Додавання/Зміна UI-компонента Додати або оновити відповідну Story у Storybook з усіма станами. Успішна статична збірка Storybook.
Нефункціональні зміни (архітектура) Оновити відповідні розділи в SSD.md або SDD.md. Оновити Traceability Matrix. Успішна збірка сайту MkDocs.
Breaking API Change Створити нову мажорну версію документації. Створення нового Version tag у Git.

2. Критерії готовності (Checklist)

Перед тим як Pull Request може бути злитий у гілку main, він має відповідати наступним критеріям: - [ ] Що має існувати: Для кожної нової функції або зміни інтерфейсу існує відповідний опис в Markdown, YAML або Stories. - [ ] У якому вигляді: Markdown не містить розбитих посилань, специфікація API відповідає стандарту, а UI-stories відображають як успішні, так і помилкові стани. - [ ] Де це перевіряється: Усі лінтери документації та генератори статичних сайтів у CI-пайплайні завершилися зі статусом Success.
